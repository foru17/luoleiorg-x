# 文章分享卡片功能

## 目标

为文章页增加一个可自动生成的分享卡片能力。用户在 `AI 摘要` 旁点击 `分享` 后，弹窗会先生成当前文章的 PNG 分享图，然后支持复制图片、复制链接或下载 PNG。

## 背景

文章页已经具备标题、封面、AI 摘要和 canonical URL 等结构化数据，适合进一步做成高质量的分享卡片，降低用户二次整理截图和文案的成本。

## 实现

### 前端交互

- 在 `src/components/article-ai-summary.tsx` 中把原有的 `AI 摘要` 入口扩展成按钮组。
- 新增 `src/components/article-share-dialog.tsx` 作为分享弹窗：
  - 保持单列结构，只展示一张 PNG 预览
  - 只保留 `复制图片`、`下载 PNG`、`复制链接` 三个动作
  - 让用户首屏就能完成操作，不需要滚动和截图

### 海报生成

- 新增 `src/app/api/share/article/[slug]/route.ts`
- 路由根据文章 `slug` 动态读取：
  - 标题
  - AI 一句话摘要 / OG 描述 / excerpt
  - 文章地址
  - 封面图
- 新增 `src/lib/share/article-share-card.ts`：
  - 对标题、摘要和地址做固定行数截断，避免排版错位
  - 输出稳定的 SVG 源图，所有图片素材都在服务端转成 data URI 内联
  - 顶部展示博客 Logo 与作者头像
  - 主体展示文章缩略图、标题和一句话摘要
  - 底部展示文章地址和二维码
- 弹窗在客户端把 SVG 源图一次性转成 PNG：
  - 预览、复制和下载都复用同一张 PNG
  - 避免浏览器直接预览 SVG 时出现排版漂移

### 设计方向

- 弹窗走轻量单列，不再堆额外说明区和信息面板
- 海报走更克制的 editorial poster 风格，优先标题层级和留白
- 生成结果优先用于复制 PNG，而不是让用户自己裁剪截图

## 结果

- 文章页 AI 摘要旁新增分享按钮
- 弹窗内可直接预览文章 PNG 海报
- 支持直接复制图片
- 海报支持下载 PNG
